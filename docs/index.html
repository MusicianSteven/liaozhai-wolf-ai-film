先用一分钟讲清楚这是个什么作品。故事就是课本原文:屠夫卖肉晚归,两只狼一路尾随,骨头喂完了狼还跟着;他退到麦场柴堆下拔刀对峙,一只狼假装睡觉引诱他,另一只绕后打洞偷袭,屠夫看破骗局,先劈假寐之狼,再断隧入之狼。几个数字请大家记一下:十六个镜头、三十三次生成任务、三分十七秒成片。注意"三十三比十六"这个比例——平均每个镜头两个版本,这个迭代过程正是今天要讲的方法论核心。还有一个反直觉的数字:全片百分之六十的时间没有配乐,这是刻意的,后面会讲到"静默"在这部片子里是叙事手段。
二三重挑战

CHALLENGES

为什么这是 AI 视频的困难模式

水墨风格的一致性

生宣质感、笔墨浓淡、留白构图,要跨 33 次独立生成保持同一张"纸"的气质——风格漂移一次,整卷画就断了。

多主体空间调度

一人两狼,狼的数量、体型、距离每一项都是生成模型的重灾区;原文"眈眈相向""犬坐于前"对空间关系有精确要求。

文言默片叙事

无旁白、无对白,叙事全靠画面表演与字卡节奏;"假寐"骗局要同时骗过屠夫和观众,表演差一点戏就塌。

最难的一镜:"其一犬坐于前"——狼要像狗一样坐下,但绝不能变成狗。这五个字,花了六个版本。

为什么说这是困难模式?第一是风格。水墨不是滤镜,是笔墨语言——三十三次生成,每一次模型都是从零开始画,凭什么保证是同一张纸、同一支笔?第二是空间。AI 视频最不擅长的就是多主体的数量和位置控制,而这个故事恰恰是"一人两狼"的空间戏:狼跟多远、停多远、坐在哪,原文都写死了。第三是叙事形态,我选了默片——没有旁白兜底,画面演不出来就是演不出来。给大家举个例子:原文"其一犬坐于前",狼像狗那样蹲坐——但它必须还是狼。这五个字我们生成了六个版本才过,后面会展示这个迭代。
三技术底座

GMI CLOUD × HAPPYHORSE 1.1

底座:HappyHorse 的四个关键能力

r2v · 多参考图生视频

每镜携带 1–5 张参考图(风格锚+角色卡+场景卡),是水墨风格与角色跨镜一致的基石。建置镜、复杂调度镜全靠它。

i2v · 首帧像素锁定

首帧即合同——比例、距离、状态在首帧里画死,文字只驱动动作。链式尾帧衔接(上镜尾帧=下镜首帧)让七个镜头共享同一空间。

随片音效(audio on)

风声、脚步、兽足、刀击随画面一次生成,默片形态下省掉整层拟音;个别缺陷用负面词+外部素材修复。

工程可靠性

时间码分段("第一刀在 2 秒内")可靠执行;request_id 全程可查、超时可 resume 续接不重复计费、1080P 直出。33 次任务全部落盘可复现。

整个项目搭在 GMI Cloud 的 HappyHorse 1.1 上,我想强调它四个对创作真正管用的能力。第一,r2v 多参考图——这是一致性的根,我给每个镜头喂一到五张参考图,风格锚永远占第一张。第二,i2v 首帧锁定——首帧就是像素级合同,这个能力撑起了我整套"源头治理"方法,也让镜头能像接力棒一样用尾帧串起来。第三,它生成视频时可以直接出环境音效,做默片等于省掉了一整层拟音工作。第四,工程性:每个任务有 request_id 可以查、轮询超时可以续接而且不重复扣费、1080P 直接出片。三十三次任务的日志和参数,我全部存档在 GitHub 上,每一条都能复现——这也是我们的模型使用证明。

整条管线八步,我快速过一遍,重点讲次序为什么这么排。最先做的不是写提示词,是读文本——这篇文章的收尾"禽兽之变诈几何哉,止增笑耳",定了全片的基调是说书人的冷笑,不是恐怖片。这个判断影响了后面每一步:配乐、节奏、甚至字卡上不上"假寐"这两个字。然后是分镜、资产、提示词、生成、质检、音频、合成。请注意第五步"生成纪律"里有个东西叫审核门禁——任何一次生成 API 调用之前,完整的正负提示词、时长、预期内容都要先过人工审核。这听起来慢,实际上是省钱省时间的:它逼着你把每一次生成都当成有成本的决策,而不是抽卡。
五方法之一

METHOD 01 · CONSISTENCY

参考卡体系:一致性是"发"出来的

  • 风格锚恒占 [Image 1]:一张生宣水墨基准图,33 次生成从不缺席——风格不是每次重新描述,是每次重新"看见"。
  • 角色卡按叙事状态分版本:屠夫有挑担/正身/空筐三态;双狼卡锁定"深墨前狼、额白后狼"的身份特征。
  • 角色行为与原文互锁:S03 深墨得第一块骨 ↔ S09 额白径去、深墨犬坐 ↔ S12 额白隧入——改一处,必查另一处。

发现·参考图污染:骨头明明喂完了,筐里却凭空冒骨——根因是角色卡上画着骨头。参考图上的任何元素都可能被继承,文字压不住。解法:为"骨已尽"之后的剧情专门重制空筐卡。资产要跟着叙事状态走。

第一个方法,参考卡体系。核心一句话:一致性不是靠文字描述出来的,是靠参考图"发"出来的。我建了一套卡:风格锚图永远占第一张参考位,屠夫按剧情状态分三个版本的卡,两只狼一张卡锁定各自特征——前狼通体深墨,后狼额头有白斑,这个特征还和原文的角色分工互锁:哪只狼吃了第一块骨头、哪只狼假寐、哪只狼打洞,全片必须对得上。这里分享一个踩坑发现,我叫它"参考图污染":有个镜头,骨头剧情上已经喂完了,生成出来筐里却一直冒骨头,提示词怎么禁都禁不住。最后发现根因是参考卡上画着骨头——参考图上的任何元素都可能被模型继承,文字压不住图。解法很朴素:重画一张空筐卡。教训是:资产必须跟着叙事状态分版本。

发现·切镜是掩护:紧跟一次切镜(近景切远景)的镜头,r2v 的"语义近似开场"就够用;而插入特写(cutaway)让宽镜头时空冻结在上一帧——传统剪辑语法,恰好是 AI 场景不连续的天然遮羞布。

第二个方法解决空间连续。从屠夫退到柴堆下、到两狼对峙、到暴起杀狼,这七个镜头必须发生在同一个麦场——AI 每次重新生成场景必然漂移,怎么办?答案是链式尾帧:上一镜的最后一帧,直接作为下一镜 i2v 的首帧,空间就像接力棒一样传下去,做到像素级连续。中间有个特写镜头很有意思——狼假寐的大特写。它不参与空间链,是从宽镜头里裁切放大出来的"插入镜"。这其实是电影剪辑一百年前就有的 cutaway 语法:切进特写的时候,宽镜头的时空就冻结了,切回来从冻结处继续。传统剪辑语法恰好成了 AI 场景不连续的天然掩护——这是我这个项目里最喜欢的一个发现:AI 视频的解法,很多时候藏在电影史里。
七方法之三

METHOD 03 · FIRST-FRAME ENGINEERING

源头治理:比例交给像素,动作交给文字

文字反复压不住的(狼的体型、距离、"死狼+半入之狼"的复杂状态),一律改为定制首帧——把约束画进像素,提示词只写动作。以全片最复杂的 S12「转视积薪后」为例:

发现·i2v 的两条边界:① 管不住"中途入画的新主体"数量(首帧无狼+文字锁双狼,仍出三只——要新主体入画,回 r2v+角色卡);② "保持某状态"类约束会失效——正是这镜的 v1/v2:狼把全身挖进了柴堆。解法只有:状态画进首帧+时长压到 6 秒。

第三个方法,我叫它源头治理,一句话:比例交给像素,动作交给文字。屏幕上是全片最复杂的一镜,S12"转视积薪后":画面里要同时成立三重状态——前景一只死狼、柴堆里一只只露后半身的狼、还有屠夫走近后的透视比例。这三样东西用文字描述,前两版全部失败:狼要么把全身挖进柴堆,要么人走近了体型不变,状态根本锁不住。最后的解法是把这三重状态全部交给 Nano Banana 画进首帧——左边这张图,死狼、半入之狼、按透视缩小的狼身,全在像素里定死;然后交给 HappyHorse 的 i2v 起播,提示词只剩动作:拱动、收刀、放轻脚步。右边可以点击播放六秒成片,大家看状态是零污染的。同时诚实讲 i2v 的两条能力边界,都是实测出来的:第一,它管不住中途入画的新主体数量,首帧没狼、文字写两只,还是进来三只,这种情况必须回 r2v 加角色卡;第二,"保持某状态"的持续约束迟早失效,这镜的前两版就是证据,唯一可靠的办法就是状态进首帧,再把时长压短到六秒。
八方法之四

METHOD 04 · CONSTRAINT LANGUAGE

模型听得懂的,是构图,不是尺子

语义单位 · 失败

"狼与人至少保持三个狼身长的距离"
"两狼在 30 米外缀行"

构图语言 · 成功

"狼停在画面右侧三分之一处,人与狼之间隔着超过半个画面宽度的空地"

  • 负面词工程:狼数量六连禁(第三只狼/狼群/多只狼…)+ 防狗态八连禁(dog/垂耳/摇尾巴…)+ 全片恒禁(血色/文字水印/写实人脸/狗叫)。
  • 时间码节拍可控:"第一刀必须在 2 秒内出现"被精确执行——节拍写死,比写"尽快"有效。

发现·名词会显形:提示词写"洞口旁的狼尸",柴堆上就真的生成了一个洞。删诱导名词 + 负面显式禁,双保险。

第四个方法关于提示词本身:模型听得懂的是构图,不是尺子。屏幕上两张截图来自同一个镜头、同一个时间点。左边第三版,提示词老老实实写着"至少保持三个狼身长的距离"——结果狼一路奔到画面中央,离屠夫只剩一个狼身,原文的"狼不敢前"完全垮掉;模型对米、对狼身长这类物理单位没有稳定概念。右边第四版只改了一句话,把尺子换成构图:"狼停在画面右侧三分之一,人狼之间隔超过半个画面宽的空地"——大家看,狼稳稳停在右缘,中间大片留白,对峙感回来了,顺带还成全了水墨的留白美学。这就是我说的:构图语言才是模型真正的母语。负面词这边我建了两套组合拳:锁狼的数量用六连禁,防止犬坐变狗用八连禁,再加全片恒定的禁项——不能有血、不能有文字水印、屠夫的脸不能变写实、音效里不能混狗叫。最后一个好玩的发现:提示词里的名词会"显形"。我写"洞口旁的狼尸",结果柴堆正面真的凭空生成了一个洞——模型把每个名词都当成绘制清单。所以修复方式是删掉诱导性名词,再用负面词显式禁止,双保险。
九质检闭环

QA LOOP

33 次生成,每一次都有验收单

  • 审核门禁:每次生成前,正负提示词+时长+预期内容先过人工审核——生成是决策,不是抽卡。
  • 探针先行:风格镜、高危镜(犬坐/暴起)先跑低成本探针,验证过再量产。
  • 抽帧自检清单:数狼 → 量距离(画内可度量)→ 验外观漂移 → 查道具状态 → 查画面文字污染。
  • 裁剪即修复:结尾狼看镜头?裁掉 1.5 秒,不重生成——后期剪刀是零成本的"最后一版迭代"。
方法讲完了,讲纪律。三十三次生成,每一次都过同一套流程:生成前有审核门禁,高危镜头先跑便宜的探针版本,出片后先抽帧自检——数狼的数量、量人狼距离、盯着屠夫的脸看有没有漂移、查道具状态、查画面上有没有冒出文字——自检过了再人工终验。屏幕上是"犬坐"镜头最后两版的真实对比:左边第五版,狼坐下之后活脱脱一条家犬,神态都温顺了;右边第六版,加了防狗态八连禁,同时正面强调尖吻、竖耳、鬃毛、直尾,狼相和冷酷的眼神就回来了。还有一条特别想分享:剪刀也是迭代工具。第六版结尾狼突然看镜头,怎么办?不重新生成——直接裁掉最后一秒半。后期裁剪是零成本的最后一版迭代,别什么问题都用重生成解决。
十规则沉淀

AGENT SKILLS

Skills:教训只犯一次,规则全局复利

AI agent 有一份随项目生长的"项目章程"(SKILL.md)——每个踩坑沉淀成规则条文,此后每一镜生成前自动遵守。这是 agent 构建项目与"逐镜手工调 prompt"的核心效率差。

暮色 · 光线时间轴

全片"黄昏(S01-05)→暮深(S06-10)→入夜月升(S11-15)"写成规则,每镜提示词自动标注当前时刻——一次设计,15 镜零跳日。

狼与人的体型 · 距离

S09 踩坑后沉淀:"肩高≈身高 1/3 + 视觉高度逐镜按透视计算"+ 距离分级表(缀行≥30 米/对峙三个狼身长)。S07/S08/S11/S12 直接继承,不再逐镜试错。

行进方向圣经

屠夫行进=画面右、狼永在人后、对峙段反转为人左狼右——一条方向规则写进章程,16 镜镜头方向零混乱、零返工。

审核门禁

S02 尾段一次擅自简化分镜,升级为最高优先级流程规则,贯穿其后全部生成任务——流程本身也在迭代。

复利证据:项目后段的 S00 / S10 / S13 / S15 全部一版通过——不是运气,是前段踩的坑都已变成规则,agent 在每次动笔前自动"翻章程"。人只判一次,机器执行 N 次。

这一页讲 agent 工作方式里我认为最被低估的机制:Skills,可以理解为 agent 的项目章程。每次踩坑,我们不是修完这一镜就算了,而是把教训写成一条规则放进 SKILL 文件,agent 之后生成任何镜头之前都会自动遵守它。举三个真实例子。第一,暮色:全片光线"黄昏到暮深到入夜月升"写成时间轴规则,之后每一镜的提示词自动标注当前时刻,十五个镜头零跳日——这个问题我们一次都没踩过,因为规则先行。第二,狼和人的相对大小:在犬坐镜头上踩过坑之后,沉淀成"肩高约等于身高三分之一、视觉高度逐镜按透视计算"加一张距离分级表,后面四五个镜头直接继承,再没为体型返工过。第三,行进方向圣经:屠夫回家永远朝画面右走、狼永远在人身后、到麦场对峙才反转成人左狼右——一条规则,十六个镜头没有一次方向穿帮,这种全局约定靠逐镜手写提示词是必然会漏的。第四,连流程本身也在迭代:有一次生成擅自简化了分镜,我们把"生成前必须人工审核"写成最高优先级规则,之后再没发生过。复利的证据就是:项目后段的四个镜头全部一版通过。教训只犯一次,规则全局生效——这就是用 agent 做项目,时间效率随项目进行越来越高的原因。

十二传统对照

VS TRADITIONAL

差异化:把导演意图工程化

维度传统水墨动画(如《山水情》)本片 AI 管线
团队原画/动画/描线/上色/摄影,多工种团队1 人 + AI agent
周期数月至数年数日(净制作)
成本人力密集,水墨晕染逐帧手工33 次生成任务,约降两个数量级
迭代改一镜=重画,代价极高改 prompt/首帧重跑,小时级
一致性靠画师功力,不可转移参考卡+首帧工程+负面词,可复制的工程方法
前期验证分镜稿/样片昂贵探针机制,小时级验证演出方案

定位:这不是"AI 替代画师",而是把分镜、空间、表演、节拍全部转译成模型可执行的约束语言,再用质检闭环收敛——导演意图的工程化。

跟传统方法比一比。一九八八年的《山水情》是中国水墨动画的巅峰,那背后是上海美影厂整个团队、以年计的工期——水墨晕染要逐帧手工完成,这也是为什么这个片种后来几乎绝迹:太贵了。同样是水墨动画,这部《狼》是一个人加一个 AI agent、几天、三十三次生成任务。但我想强调,重点不在"便宜了多少",在最后一行那个定位:这套方法不是 AI 替代画师,而是把导演意图工程化——分镜怎么写、空间怎么锁、表演怎么约束、节拍怎么卡,全部转译成模型能执行的语言,再用质检闭环去收敛结果。画师的功力是不可转移的个人天赋,而这套方法是可复制、可交接、可规模化的工程资产。这是两种生产范式的差别,不是快慢的差别。
十三诚实局限

KNOWN LIMITS

还没解决的四件事

持续状态会失控

"保持半入洞口"这类约束,i2v 迟早违背;目前只能靠首帧+短时长压制,治标不治本。

镜内慢漂移

长镜头里柴堆结构缓慢变形;12 秒以上、多主体交互仍是高风险区。

帧级音画同步

刀劈中狼首的那一帧,生成端无法保证——鼓点贴刀,靠的是后期波形对帧。

拼接点痕迹

个别镜头由两段生成拼接,交界处有轻度跳变;字卡为硬切,未做逐字墨现。

在展望未来之前,先诚实交代还没解决的问题——这些也原样写进了参赛文档。第一,持续状态失控,就是刚才那只该半入洞口的狼,现在的压制手段治标不治本。第二,镜内慢漂移:柴堆在长镜头里会缓慢变形,我们评估后接受了,但十二秒以上的多主体长镜头依然是高风险区。第三,帧级音画同步:刀劈中狼首的那一帧,生成端给不了保证,鼓点能贴上去靠的是后期用波形扫描逐帧对位。第四,两段生成的拼接点有轻度跳变,字卡也还是硬切。我把局限讲得这么细,是因为下一页的每一条展望,都是从这些具体的痛点里长出来的,不是凭空畅想。
十四发展展望

OUTLOOK

从"能生成"到"可导演、可量产"

模型侧

状态锁定、多主体身份绑定、镜头语言原语(景别/机位)成为一等公民参数;尾帧-首帧衔接官方 API 化。

管线侧

VLM 自动抽帧质检替代人工数狼;参考卡+prompt 模板系列化——聊斋其余篇目即插即用,单人可维护周更 IP。

应用侧

文言课文可视化(《狼》即初中课文);默片零语言门槛出海;影视 previz 创意验证层——先 AI 验证分镜,再决定手工精制。

一句话愿景:当空间、状态、节拍都成为可声明的参数,AI 视频就从素材生成器,变成导演的执行部门。

展望分三层。模型侧,我最期待的是把这次靠技巧绕过去的东西变成原生参数:状态锁定、多主体身份绑定、景别机位这些镜头语言原语,以及把尾帧接首帧做成官方 API——今天我手工做的链式衔接,应该是平台的一个开关。管线侧,质检环节完全可以自动化:用视觉语言模型替我数狼、量距离、盯脸,人只做美学终审;参考卡和提示词模板沉淀下来,聊斋剩下的四百多篇故事就是即插即用的量产线,一个人维护一个周更的水墨聊斋 IP 是现实的。应用侧,最落地的是教育——《狼》本身就是初中课文,这套形态可以覆盖整个文言文课本;默片形态天然零语言门槛,水墨美学又是最有辨识度的文化输出;对影视工业,它是前期验证层,先用 AI 验证分镜和演出,再决定要不要投入手工精制。一句话愿景:当空间、状态、节拍都成为可声明的参数,AI 视频就从素材生成器变成导演的执行部门。
